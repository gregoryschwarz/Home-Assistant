rendering. The custom component does not inject CSS variables or theme tokens.

Reference values (HA defaults):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | HA `--primary-background-color` | Settings page background |
| Secondary (30%) | HA `--card-background-color` | Pipeline config card, form fields |

## Interaction Contract

This section documents the step-by-step operator interaction for Phase 4.
The executor follows this sequence during the single plan (04-01).

### Step 1 — Install Wyoming Add-ons (HA OS / Supervised)

**Interaction:** Settings > Add-ons > Add-on Store > search "Whisper" > Install > Start > Auto-start on
**Interaction:** Settings > Add-ons > Add-on Store > search "Piper" > Install > Start > Auto-start on
**Fallback (HA Container/Core):** Run Wyoming Docker containers manually; skip add-on store steps.

### Step 2 — Register Wyoming Protocol Integration

**Interaction:** Settings > Devices & Services > Add Integration > search "Wyoming Protocol"
- For Whisper: host = `localhost`, port = `10300`
- For Piper: host = `localhost`, port = `10200`
**Expected outcome:** "faster-whisper" appears in STT list; "Piper" appears in TTS list.

### Step 3 — Create Assist Pipeline

**Interaction:** Settings > Voice Assistants > Add Assistant
Fill fields per Copywriting Contract table above.
Click "Save".
**Expected outcome:** Pipeline "Mon assistant HA" appears in assistant list.

### Step 4 — Validate via WebSocket Smoke Test

**Interaction:** Execute WebSocket call from RESEARCH.md Code Examples §Smoke Test.
**Expected outcome:** Event `intent-end` received with non-empty `speech` field matching agent response.

### Step 5 — Run Unit Tests

**Command:** `pytest tests/test_voice_pipeline.py -x -q`
**Expected outcome:** All 4 tests pass (VOICE-01 × 2, VOICE-02 × 2).
