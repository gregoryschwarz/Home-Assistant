# Plan 06-01: Habit Context Injection into ClaudeClient

## Objective

Étendre `ClaudeClient.async_complete` avec un paramètre optionnel `habits=None` qui injecte les habitudes pertinentes dans le message utilisateur envoyé à Claude. Câbler la récupération des patterns depuis `conversation.py` avec filtrage contextuel (entité mentionnée OU heure ±2h).

## Context

- @custom_components/ha_ai_agent/claude_client.py — `async_complete(text, entities)` à étendre
- @custom_components/ha_ai_agent/conversation.py — point d'appel, accès à `pattern_detector` via `hass.data`
- @custom_components/ha_ai_agent/pattern_detector.py — `async_get_patterns()` retourne `list[dict]`
- @custom_components/ha_ai_agent/const.py — `SYSTEM_PROMPT` (ne pas modifier)
- @custom_components/ha_ai_agent/__init__.py — `hass.data[DOMAIN][entry_id]` dict avec clé `pattern_detector`

## Tasks

### Task 1: Extend ClaudeClient.async_complete with habits parameter
**type:** auto

**Goal:** Ajouter `habits: list[dict] | None = None` à `async_complete`. Quand `habits` est non vide, ajouter un bloc "Habitudes connues" après la section "Entites disponibles" dans `user_content`.

**Format d'injection (D-05):**
```
Habitudes connues (contexte personnel) :
- {entity_id} {service} le {day_name} à {hour}h ({occurrences} fois en 14 jours)
```

Où `day_name` est le jour en français (lundi, mardi, ..., dimanche) à partir du champ `day_of_week` (0=lundi, 6=dimanche).

**Changes:**
- `custom_components/ha_ai_agent/claude_client.py` — signature étendue + construction du bloc habits

**Done criteria:**
- `async_complete(text, entities)` fonctionne comme avant (rétrocompatible)
- `async_complete(text, entities, habits=[...])` injecte le bloc habits dans user_content
- `async_complete(text, entities, habits=[])` n'injecte rien

### Task 2: Add habit filtering helper in conversation.py
**type:** auto

**Goal:** Dans `_async_handle_message`, après le fallback Claude, récupérer les patterns stockés via `pattern_detector.async_get_patterns()`, filtrer les habitudes pertinentes (D-01: entity_id mentionné dans le texte OU heure dans fenêtre ±2h par rapport à l'heure actuelle), puis passer à `claude_client.async_complete`.

**Filtering logic (D-01):**
- Un pattern est pertinent si:
  1. son `entity_id` apparaît dans le texte de la commande, OU
  2. son champ `hour` est dans `[current_hour - 2, current_hour + 2]` (modulo 24)

**Changes:**
- `custom_components/ha_ai_agent/conversation.py` — import `datetime`, récupère `pattern_detector`, filtre habits, passe à `async_complete`

**Done criteria:**
- Quand le fallback Claude est utilisé, les habits pertinents sont récupérés et passés
- Si aucun habit pertinent : `habits=[]` ou `habits=None` passé (pas d'injection)
- Rétrocompatible avec les cas où `pattern_detector` n'est pas dans `hass.data` (guard)

### Task 3: Tests for habit injection
